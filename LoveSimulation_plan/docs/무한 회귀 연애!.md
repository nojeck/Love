# **\[게임 기획서\] 무한 회귀 연애 (가제)**

**작성일:** 2026년 3월 14일

**장르:** AI 음성 분석 기반 / 진지한 병맛 / 사망회귀 연애 시뮬레이션

**플랫폼:** PC (마이크 필수) / Mobile (Android/iOS)

---

## **1\. 프로젝트 개요**

* **핵심 컨셉:** 실제 여성들의 설문 데이터를 바탕으로 한 '현실적인 연애 로직'과, 실패 시 발생하는 '정신 나간 병맛 연출'이 공존하는 루프물 게임.  
* **게임의 목표:** 유저는 특정 상황(에피소드)에서 AI 여친의 질문에 음성으로 답하며 호감도를 채워야 한다. 호감도 미달 시 사건의 시발점으로 '사망 회귀'한다.

## **2\. 핵심 게임 시스템**

### **2.1. 보이스 감정 분석 시스템 (V.E.A.)**

* **STT(Speech To Text):** 유저의 발화 내용을 텍스트로 변환하여 키워드 분석.  
* **비언어적 요소 분석:** 목소리의 떨림(Jitter), 톤의 높낮이(Pitch), 성량 등을 분석하여 유저의 '진정성'을 판정.  
  * *예: 사과할 때 목소리가 떨리면 '진심' 혹은 '비굴함'으로 가중치 부여.*

### **2.2. 실제 설문 데이터 기반 가중치 로직**

* **데이터 소스:** 실제 여성들을 대상으로 한 특정 상황별 선호/비선호 반응 조사 결과.  
* **판정 메커니즘:** 유저의 답변이 설문 데이터상의 '오답'에 해당할 경우 가차 없이 호감도 하락 및 회귀 발동.

### **2.3. 혼돈 지수 (Chaos Meter)**

* **정의:** 유저가 에피소드와 관련 없는 딴소리를 하거나, 상식을 벗어난 답변을 할 때 상승하는 수치.  
* **효과:** 혼돈 지수가 일정 수준을 넘으면 여친이 기상천외한 방식으로 변하거나, '히든 병맛 루트'로 진입.

### **2.4. 기억 시스템 (Memory System)**

* **기능:** AI 여친이 이전 루프에서의 대화 내용이나 유저의 실수를 기억함.  
* **활용:** "너 아까도 그렇게 말하더니 또 그래?"와 같은 대사를 통해 유저를 압박하거나, 반복된 실수에 대해 더 가혹한 판정을 내림.

---

## **3\. 루프 및 사망 연출 (Regression)**

* **진지한 병맛 연출:** 실패 시 단순히 게임 오버가 되는 것이 아니라, 상황과 연관된 황당한 연출과 함께 회귀함.  
  * *예: "모르겠어"라고 답하면 여친이 거대화되어 유저를 압사시키거나, 우주로 던져버림.*  
* **회귀 피드백:** 회귀 직후 AI가 유저의 목소리 톤이나 답변의 문제점을 데이터 근거를 들어 냉철하게 지적 (힌트 제공).

---

## **4\. 에피소드 기획 (예시)**

### **\[Episode 01: 달라진 점 찾기의 미궁\]**

* **상황:** 카페에서 여친이 "나 오늘 뭐 달라진 거 없어?"라고 질문. (정답: 새로 산 귀걸이)  
* **주요 분기:**  
  * **정답 (귀걸이 언급 \+ 다정한 톤):** 다음 스테이지 통과.  
  * **오답 (살 빠졌어?):** '전엔 뚱뚱했냐'는 분노와 함께 헐크로 변신하여 카페 파괴.  
  * **딴소리 (오늘 날씨 좋다):** '기상캐스터 모드' 발동하여 번개로 유저 처단.  
  * **침묵/모름:** 홍수 연출로 유저 익사.

---

## **5\. 기술 스택 및 구현 가이드**

* **엔진:** Unity (3D 카툰 렌더링 권장)  
* **AI 모델:** LLM(Gemini/GPT) \+ 설문 데이터 임베딩 \+ 프롬프트 엔지니어링  
* **음성 처리:** 안드로이드 네이티브 STT / Whisper API \+ 오디오 분석 라이브러리(Librosa 등)  
* **모델링:** 러프한 3D 모델링을 사용하여 병맛 애니메이션(Scale 조절, 물리 효과)의 자유도를 높임.
